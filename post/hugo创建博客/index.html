下载安装环境

1.先下载Nodejs,Hugo(要下载hugo_extend版本)，git,go，编辑器VSCode

创建运行

hugo server -D

切换主题

首先下载主题blist，放到thems目录下，themes/blist目录结构如下:

├─archetypes
├─assets
│  ├─css
│  └─js
├─exampleSite
│  ├─content
│  │  ├─de
│  │  │  ├─blog
│  │  │  └─page
│  │  └─en
│  │      ├─blog
│  │      └─page
│  ├─layouts
│  └─static
├─i18n
├─images
├─layouts
│  ├─partials
│  └─_default
│      └─_markup
└─static
    └─fonts

然后，在文件config.toml中添加一行（如果已经存在则修改theme字段为’blist’）

theme = 'blist'

重新执行hugo serve

添加自定义CSS样式

先在static目录下创建一个css目录，里面添加一个custom.css文件,然后在config.yml文件中按需添加如下字段:

[Params]
    custom_css = ["css/custom.css"]

然后在主题blist里修改layouts/_default/baseof.html,添加如下代码:

{{ range .Site.Params.custom_css -}}
    <link rel="stylesheet" href="{{ . | absURL }}">
{{- end }}

比如，要把所有标题都替换成棕色，那么在custom.css文件里写入如下样式:

h1, h2, h3, h4 {
    color: #5C4033;
}

网站自动发布

1.安装git 2.在github创建一个项目xxx.github.io，设置github pages 3.将本地项目public目录和github远程仓库关联起来

# 初始化
git init
# 将本地public目录关联到远程仓库
git remote add origin https://github.com/piggyd3/piggyd3.github.io.git
# 暂存所有变动
git add .
# 备注信息
git commit -m '我的个人博客'
# 推送到远程仓库
git push --set-upstream origin master

4.github创建Action 在build和deploy选项里配置action,选择Deploy static content to Pages 每次提交更新后,github action会自动执行发布任务,piggyd3.github.io可以预览最新的内容

5.一键打包发布 创建一个打包发布.bat文件,内容如下:

hugo
cd public
git add .
git commit -m '更新'
git push
pause

每次改动后执行打包执行.bat就可以自动打包发布到github pages了
